| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint, PK | 内部自增主键 |
| tg_user_id | bigint, unique | Telegram 用户ID |
| tg_username | varchar | Telegram 用户名，可能为空 |
| nickname | varchar | 展示昵称 |
| invited_by_l1 | bigint, 可空 | 直接邀请人 user.id |
| invited_by_l2 | bigint, 可空 | 二级邀请人（邀请人的邀请人） |

| register_time | datetime | 注册时间 |
| last_active_time | datetime | 最近一次签到/邀请等行为时间 |
| checkin_streak | int | 当前连续签到天数 |

| activity_tag | enum | 活跃度层级：活跃/待唤醒/流失/深度流失（每日定时任务更新） |
| is_high_value | boolean | 高价值用户标记 |
| device_fingerprint | varchar | 设备指纹，用于防刷 |
| register_ip | varchar | 注册IP |
| status | enum | 正常/拉黑 |

**关键设计说明**：3级返佣不用递归查询或路径字符串，而是在用户注册时直接算好并写死 `invited_by_l1/l2/l3` 三个字段（新用户的 l1 = 邀请人id，l2 = 邀请人的l1，l3 = 邀请人的l2）。这样返佣结算时直接读这3个字段即可，不需要遍历邀请树，性能和实现复杂度都更友好。

## 二、需要你确认的两个实现细节

**1）积分过期的精度**：`points_ledger` 里每笔"获得类"流水都带 `expire_at`，理论上最精确的做法是消费积分时按"先进先出"（FIFO）从最早未过期的批次开始扣减，这样才能准确知道每一笔积分自己还剩多少、什么时候到期。但 FIFO 批次扣减逻辑实现和测试成本不低。一个简化替代方案是按"月度批次"汇总（比如"2026年7月获得的积分"作为一个整体批次，到期时间统一按当月最后一天+12个月计算），精度略低（做不到精确到某一天获得的积分），但工程量小很多。你更倾向哪种？

**2）月度排行榜推送范围**：目前设计里没明确排行榜消息模板是推送给全体会员看榜单（制造"我也想上榜"的氛围，但会增加群发量），还是只通知上榜的前N名（群发量小，但传播效果打折扣）。建议是推给全体，因为"榜单"本身就是靠"别人看到你的名字"来驱动竞争欲，只通知本人意义不大。你确认一下这个方向。

## 三、机器人交互流程图

### 1. 注册与邀请关系建立

```mermaid
flowchart TD
    A[好友点击邀请短链 domain/r/短码] --> B[短链服务查短码对应邀请人ID]
    B --> C[302跳转至 t.me/当前生效机器人?start=邀请人ID]
    C --> D[Telegram打开机器人对话, 用户点击Start]
    D --> E{该tg_user_id是否已存在?}
    E -- 是 --> F[更新last_active_time, 发送欢迎回归消息]
    E -- 否 --> G[创建users记录]
    G --> H[写入 invited_by_l1=邀请人id\ninvited_by_l2=邀请人的l1\ninvited_by_l3=邀请人的l2]
    H --> I[points_ledger写入新人礼包积分]
    I --> J[发送欢迎消息模板]
```

### 2. 签到与邀请返佣触发

```mermaid
flowchart TD
    A[用户点击/发送 签到] --> B{今日是否已签到?}
    B -- 是 --> C[提示今日已签到]
    B -- 否 --> D[写入checkins记录, 计算连续天数]
    D --> E[points_ledger写入签到积分]
    E --> F{是否为该用户首次签到?}
    F -- 否 --> G[发送签到消息模板]
    F -- 是 --> H{是否存在invited_by_l1?}
    H -- 否 --> G
    H -- 是 --> I[按配置比例给l1/l2/l3三级邀请人写入返佣points_ledger]
    I --> J[检查l1邀请人当前邀请总数是否触发5/20/100里程碑]
    J -- 触发 --> K[写入里程碑奖励points_ledger, 推送里程碑消息]
    J -- 未触发 --> G
    K --> G
```

### 3. 提现申请

```mermaid
flowchart TD
    A[用户点击 提现] --> B[展示当前可用积分/可兑换金额/客服链接]
    B --> C[用户点击 提交兑换申请]
    C --> D[创建withdraw_requests记录 状态=待处理]
    D --> E[用户实际通过客服链接联系客服兑换]
    E --> F[客服人工核实兑换完成]
    F --> G[管理员在后台将该记录标记为已完成]
    G --> H[系统写入points_ledger兑换扣除记录]
    H --> I[更新users.points_balance]
    I --> J[withdraw_requests.status=已完成]
```

### 4. 机器人容灾切换

```mermaid
flowchart TD
    A[定时健康检测任务] --> B[遍历bots表所有token调用getMe]
    B --> C{当前生效机器人调用是否失败?}
    C -- 否 --> Z[记录检测正常, 结束]
    C -- 是 --> D[标记该bot status=异常]
    D --> E[触发管理员告警通知]
    E --> F[管理员登录后台]
    F --> G[从bots列表选择一个备用机器人]
    G --> H[点击设为当前生效]
    H --> I[更新bots表 is_active切换]
    I --> J[短链跳转服务读取新的当前生效机器人]
    J --> K[所有历史短链自动指向新机器人, 无需重新生成]
```

### 5. 短链生成与域名健康检测

```mermaid
flowchart TD
    subgraph 生成短链
    A1[用户进入邀请页面] --> A2[查询domains表status=启用的域名列表]
    A2 --> A3[随机/轮询选择一个域名]
    A3 --> A4[生成唯一short_code, 写入invite_links]
    A4 --> A5[拼接完整短链 域名/r/短码 返回给用户]
    end
    subgraph 域名健康检测_定时任务
    B1[遍历domains表所有域名] --> B2[逐一检测可访问性]
    B2 --> B3{是否异常?}
    B3 -- 是 --> B4[标记status=异常, 从可分配池剔除]
    B3 -- 否 --> B5[更新last_check_time, 保持启用]
    end
```

### 6. 积分过期批处理（定时任务）

```mermaid
flowchart TD
    A[每日定时任务启动] --> B[扫描points_ledger中获得类且expire_at<=当前时间的记录]
    B --> C[计算该批次实际剩余未消费积分]
    C --> D{剩余是否>0?}
    D -- 否 --> Z[跳过, 结束]
    D -- 是 --> E[写入过期清零流水, amount=负数]
    E --> F[更新users.points_balance]
    F --> G[结束]
    H[另一分支: 扫描expire_at在7天内且有剩余的记录] --> I[触发积分到期提醒消息模板推送给对应用户]
```

### 7. 会员活跃度层级更新与唤醒推送（定时任务）

```mermaid
flowchart TD
    A[每日定时任务启动] --> B[遍历users表]
    B --> C[计算当前时间-last_active_time天数差]
    C --> D[更新activity_tag: 活跃/待唤醒/流失/深度流失]
    D --> E{是否为新进入待唤醒的用户?}
    E -- 是 --> F[创建broadcast_tasks: 使用唤醒消息模板, target=本批新进入待唤醒用户]
    E -- 否 --> G[结束]
    F --> G
```

### 8. 月度邀请排行榜结算与推送

```mermaid
flowchart TD
    A[每月初定时任务启动] --> B[统计上一自然月内各用户作为invited_by_l1的新增注册数]
    B --> C[按数量排序取前N名]
    C --> D[写入leaderboard_snapshot]
    D --> E[给前N名写入排行榜奖励points_ledger]
    E --> F[创建broadcast_tasks: 排行榜推送消息模板, target=全体会员]
```
