**关键设计说明**：3级返佣通过在注册时直接写死 `invited_by_l1/l2/l3` 三个字段实现，不使用递归查询或路径字符串。新用户注册时：`invited_by_l1 = 邀请人.id`，`invited_by_l2 = 邀请人.invited_by_l1`，`invited_by_l3 = 邀请人.invited_by_l2`。返佣结算时直接读这三个字段，无需遍历树。

## points_monthly_batches（积分月度批次表，配合"按月简化"的过期方案）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint, PK | |
| user_id | bigint, FK→users.id | |
| batch_month | varchar，如 "2026-07" | 该批次归属的自然月 |
| points_earned_total | int | 该月累计获得积分（签到/邀请/里程碑/新人礼包/排行榜奖励之和） |
| points_consumed_total | int | 该批次已被消费/过期扣减的累计数量 |
| expire_at | datetime | 固定为 batch_month 当月最后一天 + 12个月 |
| status | enum(有效,已过期) | |

**积分获得**：用户每获得一笔积分（写入 points_ledger 的正数记录），同步对当前自然月的 points_monthly_batches 做 upsert，`points_earned_total += amount`（若当月批次不存在则创建）。

**积分消费（兑换/后台调整为负）**：按批次月份升序（最早月份优先，FIFO）依次从 `points_earned_total - points_consumed_total` 有剩余的批次中扣减，直到扣完所需数量为止，同步更新每个涉及批次的 `points_consumed_total`。

**积分过期批处理**：见 04 文档，每月定时任务扫描 `expire_at <= 当前时间 且 status=有效` 的批次，对 `points_earned_total - points_consumed_total > 0` 的部分写入 points_ledger 的"过期清零"负数记录并更新 users.points_balance，随后将该批次 status 置为"已过期"。
