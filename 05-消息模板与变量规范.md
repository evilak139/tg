# 05 消息模板与变量规范

## 模板类型（对应 message_templates.type）

| 类型 | 触发场景 |
|---|---|
| 欢迎 | 新用户 /start 完成注册时 |
| 邀请 | 用户点击"邀请"菜单时 |
| 签到 | 用户完成签到时 |
| 我的 | 用户点击"我的"菜单时 |
| 唤醒 | 04文档定时任务判定用户新进入"待唤醒"层级时，自动群发 |
| 积分到期提醒 | 04文档定时任务检测到用户有积分即将到期时 |
| 月度排行榜 | 04文档每月结算后，向全体会员推送 |

## 通用变量（各类型模板均可使用）

| 变量 | 说明 |
|---|---|
| `{昵称}` | users.nickname |
| `{用户ID}` | users.id 或 tg_user_id，视展示需要 |
| `{当前积分}` | users.points_balance |
| `{邀请链接}` | 该用户对应的 invite_links 完整短链 |
| `{直接邀请人数}` | invited_by_l1=该用户 的 users 计数 |
| `{间接邀请人数}` | invited_by_l2/l3=该用户 的 users 计数之和 |
| `{连续签到天数}` | users.checkin_streak |
| `{注册时间}` | users.register_time |
| `{今日获得积分}` | 当日 points_ledger 中该用户 amount>0 之和 |

## 专属变量

| 变量 | 适用模板 | 说明 |
|---|---|---|
| `{邀请奖励值}` | 邀请 | 读取 points_config 的 invite_l1/l2/l3_points 拼接展示 |
| `{里程碑进度}` | 邀请 | 距下一个5/20/100里程碑还差多少人 |
| `{本月排名}` | 邀请 | 本月度实时或最近一次结算排名 |
| `{到期积分数}` | 积分到期提醒 | 即将过期的批次剩余积分数 |
| `{到期日期}` | 积分到期提醒 | 对应 points_monthly_batches.expire_at |
| `{本月邀请排行榜}` | 月度排行榜 | 渲染为前N名的用户昵称+邀请数列表 |
| `{身份等级}` | 我的 | users.identity_level |

## 渲染规则

- 变量缺失（如某用户从未被邀请，`{邀请链接}`拿不到上级信息但这本身是自己的邀请链接，不受影响）时，模板引擎应做合理降级展示（如显示"-"或省略对应句子），不应直接抛错导致消息发送失败
- 后台保存模板时应校验模板中出现的变量是否在允许清单内，避免拼写错误导致线上展示原始占位符文本
- 图文消息建议同时支持"仅文字"和"图片+文字"两种展示形态，兼容部分场景无图的情况
