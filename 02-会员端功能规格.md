# 02 会员端功能规格

> 对应数据表见 01-数据库设计.md；涉及消息模板见 05-消息模板与变量规范.md

## 主菜单

用户与机器人对话界面为内联键盘（Inline Keyboard），常驻四个入口：`邀请` `签到` `提现` `我的`。

## /start 处理逻辑（关注注册 + 邀请关系建立）

1. 解析 `/start {payload}`：payload 为空表示无邀请人直接关注；payload 为邀请人的 `users.id` 或对应编码表示通过邀请短链而来。
2. 按 `tg_user_id` 查询 users 表：
   - 若已存在：更新 `last_active_time`，发送"欢迎回归"文案（欢迎消息模板的一种变体，或单独判断已注册用户不重复发放新人礼包）。
   - 若不存在：
     a. 创建 users 记录，写入 tg_user_id / tg_username / nickname / register_time / register_ip / device_fingerprint。
     b. 若 payload 对应邀请人存在，写入 `invited_by_l1 = 邀请人.id`，`invited_by_l2 = 邀请人.invited_by_l1`，`invited_by_l3 = 邀请人.invited_by_l2`。
     c. 写入 points_ledger 新人礼包记录，同步 points_monthly_batches 当月累计，更新 users.points_balance。
     d. 发送"欢迎消息"模板，提示"签到、邀请均可获得积分"。

注意：此阶段**不**触发邀请返佣，返佣要等被邀请人首次签到才触发（见下方"签到"一节），这是已确认的防刷设计。

## 邀请

点击"邀请"后展示：

- 我的专属邀请短链（若该用户在 invite_links 中无记录则先生成：从 domains 表 status=启用 的域名中随机/轮询选一个，生成唯一 short_code，写入记录；若已有记录直接复用）
- 一张带二维码的邀请海报图片（图片模板后台可配置，二维码内容为上述短链）
- 当前邀请奖励规则文案（读取 points_config 中 invite_l1/l2/l3_points，用邀请消息模板渲染）
- 当前累计邀请人数、距离下一个里程碑（5/20/100人）还差多少人
- 本月排行榜当前排名（读取 leaderboard_snapshot 当前月度临时排名，若未结算可实时计算本月 invited_by_l1=我 的新增注册数排名）

对应消息模板类型：`邀请`

## 签到

点击"签到"后：

1. 判断 `checkins` 表中是否已有今日记录（按 user_id + checkin_date），若有则提示"今日已签到"。
2. 若没有：
   - 写入 checkins 记录，`streak_at_checkin` 按上次签到日期是否为昨天判断是否连续，连续则 `checkin_streak += 1`，否则重置为1。
   - 按 points_config 中的递增规则计算本次签到积分，写入 points_ledger（change_type=签到），同步 points_monthly_batches，更新 users.points_balance / last_active_time / last_checkin_date / checkin_streak。
   - **邀请返佣触发判断**：查询该用户是否是"首次签到"（即 checkins 表中此前无记录）。若是首次签到，且 `invited_by_l1` 不为空：
     - 按 points_config 的 invite_l1/l2/l3_points 给 invited_by_l1 / invited_by_l2 / invited_by_l3（若存在）写入对应返佣 points_ledger 记录（change_type 分别为邀请一级/二级/三级返佣，related_user_id=当前用户），同步各自的 points_monthly_batches 和 points_balance。
     - 对 invited_by_l1 重新统计其当前累计直接邀请人数，若达到5/20/100里程碑阈值（且此前未领取过对应档位），写入里程碑奖励 points_ledger，推送里程碑达成提示。
     - 同时更新 invited_by_l1 的 `identity_level`（注册会员→邀请达人→团队长→VIP大使，按累计邀请人数阶梯判定）。
   - 发送"签到"消息模板，展示本次获得积分、当前连续天数、当前总积分。

对应消息模板类型：`签到`（达成里程碑时另发一条里程碑提示，可复用签到模板加变量或单独做一类模板，视开发取舍）

## 提现/兑换

点击"提现"后：

1. 展示当前 `users.points_balance`，按 `points_config.exchange_rate` 换算出可兑换金额，展示客服联系方式（配置项）。
2. 提供"提交兑换申请"按钮：点击后创建 withdraw_requests 记录（status=待处理，points_amount=用户当前余额或用户输入的兑换数量视产品最终确定，exchange_amount=按当前汇率快照计算）。
3. 提示用户"已生成申请，请通过客服完成兑换"，客服人工核实后，由管理员在后台（见03文档 提现申请管理）标记完成，系统自动写入 points_ledger 兑换扣除记录并扣减 points_balance，更新 withdraw_requests.status=已完成。

风控：新账号（register_time 距今小于 `points_config.new_account_withdraw_limit_days`）申请大额提现时，创建的申请记录应带风控标记，供人工审核时重点关注（是否需要机器阻断由产品决定，MVP阶段建议先做标记提示，不做强制拦截）。

## 我的

展示字段：

- 用户名/昵称
- 当前积分（users.points_balance）
- 直接下级数量（统计 `invited_by_l1 = 我` 的 users 数量）
- 间接下级数量（统计 `invited_by_l2 = 我` 以及 `invited_by_l3 = 我` 的 users 数量之和，或分开展示二级/三级）
- 身份等级标识（users.identity_level）
- 注册时间
- 积分即将到期提醒（若存在 `points_monthly_batches` 中 `expire_at` 在近期且有剩余的批次，展示提示文案）
- 按钮：一键复制邀请链接、查看积分明细（跳转展示该用户 points_ledger 分页列表）

对应消息模板类型：`我的`
