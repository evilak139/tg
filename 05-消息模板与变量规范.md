# 05 消息模板与变量规范

## 模板类型（对应 message_templates.type）

| 类型 | 触发场景 |
|---|---|
| 欢迎 | 新用户 /start 完成注册时 |
| 邀请 | 用户点击"邀请"菜单时 |
| 签到 | 用户完成签到时 |
| 我的 | 用户点击"我的"菜单时 |
| 唤醒 | 04文档定时任务判定用户新进入"待唤醒"层级时，自动群发 |
| 积分到期提醒 | 04文档定时任务检测到用户有积分即将到期时 |
| 月度排行榜 | 04文档每月结算后，向全体会员推送 |

## 通用变量（各类型模板均可使用）

> 变量名本身已改为葡语(巴西)（面向巴西用户的机器人正文，见MessageTemplateType::commonVariables()），说明列仍用中文方便开发理解。

| 变量 | 说明 |
|---|---|
| `{nome}` | users.nickname |
| `{ID_usuario}` | users.id 或 tg_user_id，视展示需要 |
| `{pontos_atuais}` | users.points_balance |
| `{link_convite}` | 该用户对应的 invite_links 完整短链 |
| `{convidados_diretos}` | invited_by_l1=该用户 的 users 计数 |
| `{convidados_indiretos}` | invited_by_l2/l3=该用户 的 users 计数之和 |
| `{dias_checkin_consecutivos}` | users.checkin_streak |
| `{data_cadastro}` | users.register_time |
| `{pontos_hoje}` | 当日 points_ledger 中该用户 amount>0 之和 |

## 专属变量

| 变量 | 适用模板 | 说明 |
|---|---|---|
| `{valor_recompensa_convite}` | 邀请 | 读取 points_config 的 invite_l1/l2/l3_points 拼接展示 |
| `{progresso_meta}` | 邀请 | 距下一个5/20/100里程碑还差多少人 |
| `{posicao_mes}` | 邀请 | 本月度实时或最近一次结算排名 |
| `{pontos_a_expirar}` | 积分到期提醒 | 即将过期的批次剩余积分数 |
| `{data_expiracao}` | 积分到期提醒 | 对应 points_monthly_batches.expire_at |
| `{ranking_convites_mes}` | 月度排行榜 | 渲染为前N名的用户昵称+邀请数列表 |
| `{nivel_identidade}` | 我的 | users.identity_level |

## 渲染规则

- 变量缺失（如某用户从未被邀请，`{邀请链接}`拿不到上级信息但这本身是自己的邀请链接，不受影响）时，模板引擎应做合理降级展示（如显示"-"或省略对应句子），不应直接抛错导致消息发送失败
- 后台保存模板时应校验模板中出现的变量是否在允许清单内，避免拼写错误导致线上展示原始占位符文本
- 图文消息建议同时支持"仅文字"和"图片+文字"两种展示形态，兼容部分场景无图的情况
